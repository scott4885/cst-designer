## Task 2: Patient Flow Simulation

Given a schedule template, simulate what a typical patient day looks like:
- How many patients are seen?
- What's the wait time distribution?
- Which time slots are most likely to run behind?

This is an estimation engine — not real scheduling — but gives actionable insight.

### Simulation Engine (`src/lib/engine/simulate-patient-flow.ts`)

Input: one provider's schedule for one day
Output: `PatientFlowResult`

For each block:
- Assign a "typical patient count" per block type (e.g., Crown Prep = 1 patient, Hygiene = 1, Prophy = 1)
- Assign "typical duration variance" (Crown Prep: +/- 15 min, Hygiene: +/- 5 min, NP Exam: +/- 20 min)
- Simulate 100 runs (Monte Carlo, simple JS random) to get:
  - Expected patients per day
  - P50 / P90 completion time (when does the day end?)
  - Likely bottleneck slot (the block with highest variance × production impact)

### UI — Patient Flow Panel
Collapsible panel in Template Builder right column, below Optimization Advisor:
- "👥 Patient Flow Estimate" header
- Cards per provider: Expected patients | P50 end time | P90 end time
- Warning if P90 > scheduled end time + 30min: "⚠️ Dr. Smith's schedule may run long on 1 in 10 days"
- "Bottleneck: Crown Prep at 2:00 PM — high variance procedure"

Keep it simple — no complex UI. 3–4 cards, clear numbers, actionable warnings.

## Task 3: Weekly Schedule Report (PDF-ready)

A printable, professional weekly schedule summary for practice managers.

Route: `/offices/[id]/report`

### Report Layout (single HTML page, print-optimized):
1. **Header**: Office name, week of [date], generated date
2. **Executive Summary**: Total scheduled production, quality score, goal achievement %, provider count
3. **Daily Schedule Cards**: one card per working day (Mon–Fri)
   - Provider schedule summary (not full grid — block list format)
   - Daily scheduled production vs. goal
   - Clinical warnings for that day (if any)
4. **Procedure Mix Summary**: Current vs. target mix table
5. **Provider Performance Table**: provider name | role | daily goal | scheduled production | gap | score
6. **Footer**: Generated by Schedule Template Designer

### Print button: Opens browser print dialog (landscape, CSS @media print)
### "Download Report" button: Triggers window.print() with filename hint

Style: clean, professional — similar to Project Fusion report aesthetic (white bg, clear typography, no flashy colors).

## Task 4: Schedule Change Notifications (In-App)

A simple in-app notification system for schedule-related events.

### Notification Types:
- "✅ Schedule saved — Monday Week A"
- "📋 Template cloned to 5 offices"
- "⚠️ Clinical rule violation detected — 3 warnings"
- "💡 Optimization suggestion: +8 pts available"
- "🔄 Version restored — Monday 2:15 PM"
- "📊 Quality score improved: 72 → 81"

### Implementation:
- `src/lib/notifications.ts`: `NotificationStore` (Zustand slice)
  - `addNotification(type, message, link?)` — max 20 stored, oldest pruned
  - `markRead(id)`, `clearAll()`
  - Notifications persist in localStorage

- **Bell icon** in app header (top right): shows unread count badge
- Click → dropdown showing recent notifications (last 10)
- Each notification: icon + message + relative time ("2 min ago")
- "Mark all read" + "Clear all" buttons

- Wire into existing save, clone, restore, validation, quality score events throughout the app
